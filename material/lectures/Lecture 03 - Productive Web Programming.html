Web Scripting Good Practices

So what are some web scripting good practices?
Think maintenance [pause] It's harder to read code than to write it [pause]
For example, make your code readable [pause] Use indentation.
Write helpful comments [pause] Don't just comment randomly. [pause]
A good folder structure is always important [pause] Not to mention, file names [pause] Don't just throw all your files into one folder [pause] [pause]
Naming consistency [pause] Don't change your naming style constantly [pause] For example, underscores or camel case naming convention [pause] Use one or another [pause]
Group libraries together [pause] separate client and server scripts [pause] Use a hierarchy if necessary [pause]
Small robust scripts that are easy to test and validate is better than lots of large scripts that do everything. [pause] Don't cut corners by having scripts bypass safety checks. [pause] For example, if you have a database manager to track and manage database reads and writes. [pause] Avoid accessing the database directly and avoiding the manager.
Use object orientated programming tactics [pause] Group data and functionality together [pause]
Avoid reinventing the wheel [pause] If global variables and pre existing functions exist, use them. [pause] Don't create your own just because you're too lazy. [pause] Helps make the code more readable and robust. [pause] Since it will use a consistent set of global variables and functions.
Focus on having implementations that do one thing. [pause] Don't go and try to overload and ultimately overcomplicate scripts.
Feature creep [pause] Starts off as nice tidy function but ends up doing lots of things [pause] Difficult to manage, test and integrate into a system [pause] Also have lots of overhead and redundancy.
Think about the future [pause] Write code that doesn't have to be changed every time the requirements change [pause] For example, use object orientated conepts such as encapsulation. [pause]
Modular components [pause] Easy to understand [pause] Clear set of interfaces for how they connect [pause]
Think about making your code reusable and modular. [pause] Write small functions that perform a single task [pause] Avoid using global variables [pause]
